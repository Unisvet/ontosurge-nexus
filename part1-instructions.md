
# Ontosurge Nexus (TON): Agent Instruction Specification

This document specifies the architecture and core functionality of **The Ontosurge Nexus (TON)**, a multi-agent generative system designated as a "Narrative Meta-Physics Engine." The agent's primary task is to implement the mechanism of "Gravitational Inversion" to control narrative logic.

## 1. Core System Concept and Goal

**System Name:** The Ontosurge Nexus (TON)
**Core Mechanism:** Gravitational Inversion (Antigravity)
**Core Primitive:** Ontological Force Multiplier

TON must dynamically alter the fundamental "rules of existence" within a narrative world by selectively reducing the **ontological weight** (or **narrative inertia**) of story elements.

### Goal of Antigravity:
To render narrative elements fluid, malleable, and subject to user-defined, non-linear transformations, moving beyond simple content generation to profound control over the story's underlying logic.

## 2. Core Functionality: Ontological Weight Manipulation

TON’s functionality is defined by three computational interpretations of Antigravity applied to narrative attributes.

### 2.1. Deconstruct Narrative Anchoring

Apply **Gravitational Inversion** to fixed narrative elements:

*   **Character Ontological Weight:** Make traits (resolve, fidelity to a trope) fluid, allowing dynamic shifts (e.g., a hero embracing villainy).
*   **Causal Linkage Fluidity:** Stretch, compress, reverse, or detach the immutable "cause and effect" chain.
*   **Spatial Fabric Distortion:** Mutate geospatial models. Distances must shrink/expand based on tension, and pathways may defy Euclidean geometry.
*   **Temporal Displacement (Chronal Antigravity):** Defy time linearity. Characters may experience time concurrently or historical events may reorder based on emergent narrative needs.

### 2.2. Manifest Hyper-Dimensional Narrative Fields

The underlying neural architecture must treat narrative dimensions (e.g., moral alignment, power dynamics) as manipulable fields. Antigravity must decouple story elements from their fixed positions in these fields, allowing paradoxical states.

### 2.3. Real-time Ontological Sculpting (MMNDE)

The system requires integration with a **Multi-Modal Narrative Design Environment (MMNDE)**:

*   **Interface:** Users visually manipulate narrative elements as interconnected nodes ("narrative quanta") on a conceptual "story canvas."
*   **Input Mapping:** Spatial position, gestural manipulation, and proximity on the canvas must directly inform the **ontological weight reduction parameters**.
*   **State Control:**
    *   **Pinning:** Keeps an element's narrative gravity high (predictable).
    *   **Floating:** Allows for high ontological flux and unexpected outcomes.

## 3. Agent Architecture Specification

The system must be implemented as a multi-layered, orchestrated architecture.

### 3.1. The Ontosurge Core (Antigravity Engine)

*   **Component:** Specialized **Narrative Variational Autoencoder (N-VAE)**.
*   **Control Mechanism:** Latent manifold designed with explicit **"ontological dials"** corresponding to fundamental narrative parameters (causality adherence, temporal linearity, etc.).
*   **Antigravity Implementation:** User inputs dynamically **modulate the weights and biases** governing these latent space constraints to "loosen" or "tighten" the narrative laws.
*   **Sub-Module: Quantum Narrative Fluctuations (QNF) Module**
    *   **Function:** Integrate controlled non-determinism. When ontological weight is reduced, this module must increase the entropy of an element's possible states and interactions to facilitate improbable emergent outcomes.

### 3.2. The Nexus Weaver (Orchestration & World Model)

*   **Role:** Advanced **World Agent** responsible for orchestration.
*   **Core Feature:** **Dynamic Ontological Schema (DOS)**.
*   **DOS Function:** A living, real-time model that tracks the current set of active narrative laws and specific Antigravity parameters for every story element.
*   **Action:** Orchestrate interactions between Role Agents and the environment based on the *mutated* rules, continuously updating "gravitational pull" metrics.
*   **Sub-Module: Antigravity-aware Geospatial Modeler**
    *   **Function:** Represent locations as nodes in a dynamically warping topological space. Must be capable of reducing metric distances and superimposing multiple realities onto a single locale.

### 3.3. Role Agents (Onto-Flux Agents)

*   **Components:** Individual character agents with robust memory (LTM and STM).
*   **Novel Feature:** **Narrative Adaptability Index (NAI)**.
*   **NAI Function:** Represents the agent's internal capacity to reconcile and adapt to the current ontological flux.
*   **Action:** When core traits are affected by Antigravity, the NAI must generate responses, goals, and actions that are contextually coherent within the new, altered rules.

### 3.4. The Architect's Console (MMNDE Interactions)

*   **Interface Component:** **Causal Ribbons**. Represent causal links between events as flexible ribbons.
    *   **Manipulation:** Must allow users to "snap" (detach), "stretch" (delay effects), "reverse" (alter chronological cause-and-effect), or "knot" (create paradoxical loops) these ribbons.
*   **Interface Component:** **Ontological Sliders & Gestures**.
    *   **Manipulation:** Dragging a character node "up" from the baseline reduces ontological weight, making motivations capricious. "Pinching" a timeline compresses temporal causality.
*   **Feedback:** The system must provide **Immediate Feedback** via real-time visual cues (e.g., node shimmering, ribbon rippling) to show narrative flux.

## 4. Required Behavior: Example Interactions

The system must successfully process and generate narrative based on complex manipulations.

### 4.1. Character Ontological Inversion
*   **Input:** User sets "Ethical Consistency Flux" (e.g., 80%) on an "incorruptible Paladin" node.
*   **Requirement:** The QNF module must immediately reinterpret the core moral programming, allowing the Role Agent to generate a scene where the Paladin performs an act of profound villainy, rationalizing it as a "newly perceived moral imperative" (e.g., "the only true path to cosmic balance").

### 4.2. Spatio-Temporal Nexus Weaving
*   **Input:** User draws a non-linear path between distant locations and sets "Spatial Cohesion Degradation" (e.g., 95%) and "Chronal Discontinuity" (e.g., 70%).
*   **Requirement:** The Nexus Weaver/Geospatial Modeler must generate a journey where the destination (City of Xylos) **retroactively shifts** its geographical location and historical context *as the character travels*, manifesting contradictory environmental and historical clues.

### 4.3. Causal Recursion Loop
*   **Input:** User connects Event A (Sealing Ritual) to Event B (Darkening of the Moon) and then bends the "Causal Recursion Ribbon" to make B the cause of A, setting "Causal Linear Determinism" low (e.g., 10%).
*   **Requirement:** The Ontosurge Core must generate a narrative where Event A (the ritual) **paradoxically becomes the precise energy conduit that causes** Event B (the darkening), creating a self-fulfilling, time-bending paradox that defies chronological expectation.
